Some Math

There is some not too complicated math involved with 3D rendering. If you already know trigonometry and matrix multiplication, feel free to skip this section; if you don't, they aren't hard concepts! I'll build this on concepts from high school algebra.

Click on the following links to learn more about trigonometry and linear algebra

  • Linear Algebra

If you do know the linear algebra for transforming vectors, it is really straight forward how 3D rendering is.

My code uses the .OBJ file format; it stores a list of vertices and faces. A face is made of three vertices. By simply converting each vertex to a vector, we can transform the vectors of each face, and connect them together!

A simplified version of my code can be described as:

  1. Parse the .OBJ file by storing all the vertex and face information
  2. Convert the vertex information to a vector
  3. For each face, there are three vectors,
    • For each vector in the face, transform it using a transformation matrix such as a rotation matrix
    • Fancy rendering can do some more linear algebra to work out lighting and shading, but for a wireframe, just connect the transformed vectors in a triangle to draw the face
